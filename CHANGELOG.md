# 更新日志
## v2.1.2 (2026-06-09)

### ✨ 新增
- **东南亚地区专属分组**：新增 `sea`（🏝️ 东南亚节点）策略组，包含对马来西亚、泰国、印尼、菲律宾、越南等区域节点的自动归类。
- **自定义规则集 CDN**：在 `USER_CONFIG` 中新增 `ruleProviderCDN` 配置项，允许用户在默认 CDN (`fastly.jsdelivr.net`) 受到运营商 SNI 阻断时自行替换其他镜像源，提升规则首次拉取的成功率。

### 🔧 优化
- **正则提取逻辑重构**：重写了专线特征提取（`REGEX_LINE`）的底层逻辑，由 `while...exec` 循环替换为单行 `replace` 回调方案，规避了潜在的全局正则状态残留与死循环风险，提升处理性能。
- **干扰节点清洗强化**：机场公告正则库（`REGEX_INFO_NODE`）新增“公告、维护、不可用、失联、扣费、禁止、仅限”等常用屏蔽关键词，进一步提升无效营销节点的过滤准确度。
- **连接阻断策略升级**：将“广告拦截”与“QUIC 阻断”的底层策略由 `REJECT` 统一调整为 `REJECT-DROP`。通过静默丢包代替显式拒绝响应，促使应用更迅速地降级回退（Fallback），同时减少因频繁重试引发的设备发热。
- **防 P2P 偷跑规则完善**：各操作系统的下载工具直连规则新增了 `BitComet` (比特彗星)、`uTorrent`、`aria2c` 及 `fdm` 等主流客户端的进程匹配，有效降低代理流量被意外抽干的风险。
- **默认路由偏好调整**：基于多数用户的日常使用习惯，将 `proxyFirst`（漏网之鱼走代理）的默认状态调整为 `true`。

### 📝 修复
- **进程匹配修正**：修正了 Windows 平台下 IDM 下载器的实际进程名称（由 `IDM.exe` 修正为 `IDMan.exe`），确保相关的负载均衡规则能够正确命中。

## v2.1.1 (2026-06-08)

### ✨ 新增
- **原生/家宽 IP 优选机制**：新增 `enableResidential` 开关。自动识别节点名称中的家宽、住宅、双ISP及各地区原生宽带特征（打上 🏠 标签），并构建“🏠 家宽专用”底层策略组。
- **敏感服务防封控路由**：AI 助手（OpenAI/Gemini/Claude）、TikTok 及 PayPal 策略组会自动将“家宽节点”置于测速链/选择链的第一顺位，降低被限制或验证的几率。
- **全局 IPv6 综合管控**：新增 `enableIPv6` 总开关。实现一键控制局域网 IPv6 直连放行、TUN 栈 IPv6 路由以及 Fake-IP 的 IPv6 解析。
- **Disney+ 独立分流**：新增 `enableDisney` 开关，从通用流媒体中剥离，满足重度追剧用户的精细化跨区路由需求。

### 🔧 优化
- **节点特征库扩容**：针对 HKT、Hinet、Nuro、StarHub、Singtel 等各地区主流原生宽带/家用宽带服务商增加独立正则识别，显著提高原生 IP 的自动化提取命中率。

## v2.1.0 (2026-06-07)

### 🚀 性能重构
- **正则预编译优化**：将所有核心识别正则（入口、倍率、专线、信息节点等）提取至全局作用域并进行预编译。在处理超大规模节点列表（500+ 节点）时，显著降低 CPU 占用并提升脚本执行速度。
- **字典常量外置**：重构了地区定义与图标规则的存储方式，减少函数执行时的内存开销。

### ✨ 新增
- **地区组行为自定义**：新增 `regionGroupType` 配置，支持将地区策略组设为 `url-test`（自动测速）、`select`（手动选择）或 `fallback`（故障转移），满足不同用户的使用习惯。
- **高级测速参数控制**：新增 `testURL`、`testInterval`（测速间隔）及 `testTolerance`（切换阈值）自定义选项，允许用户根据机场质量微调自动切换的灵敏度。

### 🔧 优化
- **Unicode 属性正则匹配**：引入现代化的 Unicode 属性转义（`\p{Regional_Indicator}`）进行国旗和国家识别。相比旧版的码点计算，这种方式更准确，且能完美兼容复合表情符号。
- **清洗逻辑精细化**：重写了 `ICON_RULES` 处理逻辑，在保留关键功能图标的同时，更彻底地去除了干扰美观的杂乱 Emoji。
- **代码健壮性提升**：移除了多个正则表达式中的全局标志 `g`，规避了 JavaScript 正则 `lastIndex` 状态残留导致的节点漏识或识别跳过问题。

### 📝 修复
- 修复了某些特殊字符节点名在二次清洗时可能导致序号 `[01]` 丢失的问题。
- 修复了在部分 Mihomo 分支下，对未知的国旗图标捕获不稳定的边界情况。

## v2.0.0 (2026-06-06)

### ✨ 新增
- **用户自定义配置区**：新增 `USER_CONFIG` 对象，支持灵活开关各功能模块（AI、游戏、流媒体、社交等），无需修改代码逻辑即可定制分流行为。
- **规则集二进制格式支持**：新增 `useMRS` 开关，可选择使用 Mihomo 高性能 `.mrs` 二进制规则集，大幅提升规则加载速度（仅限 Mihomo 内核）。
- **多平台进程匹配**：新增 `osType` 配置，自动适配 Windows / macOS / Linux 的下载软件进程名（qBittorrent、Thunder、IDM、aria2c 等），精准管控 BT/P2P 流量。
- **QUIC 阻断选项**：新增 `enableQUICReject`，可选择性阻断 UDP 443 端口，优化 YouTube 等流媒体加载速度（默认关闭，避免影响游戏语音/会议）。
- **扩展分流场景**：新增 TikTok、Spotify、社交平台（Twitter/Facebook/Instagram/Discord）、加密货币、PayPal 等独立分流支持（默认关闭，按需开启）。
- **国内分流组**：新增 `enableDomesticGroup`，可选“🇨🇳 中国分流”策略组，方便海外党回国访问特定服务或国内用户精细控流。
- **信息节点保留/过滤**：新增 `removeInfoNodes`，可选择彻底过滤机场公告/流量提醒节点，或将其沉底显示，避免干扰正常选择。
- **动态捕获未知地区**：节点清洗时若遇到字典外的国旗+国家名（如 🇲🇾 马来西亚），自动归入“🌐 其他节点”，不再被丢入“🗑️ 未知识别”。

### 🔧 优化
- **策略组动态隐藏**：无可用节点的功能组（如 OpenAI、TikTok、加密货币等）自动隐藏，保持客户端界面清爽。
- **节点清洗增强**：
  - 入口城市标签、专线标签、倍率提取逻辑更严谨，减少误判。
  - 保留国旗的同时剔除其他花哨图形，节点命名更规范。
- **规则集地址统一**：所有规则集使用 CDN 加速地址（`fastly.jsdelivr.net`），并支持通过 `🚀 自动选择` 策略组下载，避免首次启动因节点未选导致规则拉取死锁。
- **核心路由模式切换**：新增 `proxyFirst` 选项，可选“直连优先”（默认，国内体验最佳）或“代理优先”（全局未识别流量走代理）模式。
- **负载均衡与下载池隔离**：低倍率节点自动归入下载池，不再参与日常测速和常规分流，防止污染正常业务。

### 📝 修复
- 修复 AI 策略组在某些地区节点缺失时重复添加 `DIRECT` 的问题。
- 修复地区匹配正则中 `CN` 误匹配 `CN2` 等专线标签的问题（通过调整匹配顺序）。
- 修复因空策略组导致内核解析报错的潜在风险。
- 修复部分节点名称中的数字被过度清理的问题（保留序号所需数字）。
- 修复 `safeList` 函数在空列表时返回 `REJECT` 导致故障转移链中断的问题，现统一返回 `["DIRECT"]`。

## v1.5.0 (2026-06-03)

### ✨ 新增
- 新增“**🗑️ 未知识别**” (垃圾桶) 机制，自动收容无地区标识或命名极其混乱的乱码节点，不参与日常测速与智能分流，确保主面板清爽。
- 新增**入口标志智能提取**功能，可自动识别并提取国内中转入口及运营商（如“深港”的“深”、“联通”等）加入标签。
- 新增针对 AI服务、学术网站、B站的**动态策略池**，智能匹配当前有效地区，自动过滤不支持的区域。

### 🔧 优化
- **节点彻底物理隔离**：将专属下载/低倍率节点（⏬）彻底从主线路中剥离，使其不再参与日常测速与常规组分流，防止污染正常业务。
- **倍率提取算法升级**：采用更严谨的正则匹配（支持中文乘号“×”），规避节点名中类似 `10x` 或 `x10G` 造成的误判。
- 强化 **BT/P2P 安全策略**：在原有规则基础上，追加 `tracker` 和 `announce` 域名关键字的双重直连保险，防测速漏网。
